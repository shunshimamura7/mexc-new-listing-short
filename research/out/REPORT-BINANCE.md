## 1. API仕様（実測）

| 項目 | 実測値 |
|---|---|
| 上場日フィールド | **`onboardDate`**（ミリ秒エポック、PERPETUALで欠損ゼロ） |
| kline レスポンス | 12要素の**配列の配列**。価格・出来高は**文字列**。`[0]`=openTime(ms), `[6]`=closeTime |
| kline 1リクエスト上限 | **1500本**（`limit=2000` は HTTP 400 `code:-1130`） |
| レート制限 | 重み方式 **2400/分**。`limit=1500` の kline は重み10（≒240 req/分が上限） |
| 認証 | 不要（public） |
| 未来行のパディング | **無し**（MEXCは有り） |
| 遡れる範囲 | **2019-09-08**（BTCUSDT）。MEXCは2020-06 |

取得は 300ms スリープ、429/5xx は指数バックオフ3回。全銘柄で 429 は一度も発生しなかった。

### contractType による分類（Binanceの優位点）

Binance は株式トークンを **`TRADIFI_PERPETUAL`** という独立した契約タイプとして明示している。
MEXC ではシンボル名に `STOCK` が含まれるかで推測するしかなかったが、Binance では**取引所自身のラベルで正確に分離できる**。

また **`status: SETTLING`（上場廃止プロセス中）が123件見える**。MEXC の `contract/detail` は現存契約しか返さず、この状態が観測できなかった。

取得窓を `onboardDate - 6h` から開始しているため、それ以前にローソクが存在する銘柄では初ローソクを取りこぼす。**該当は `ETHBTC` の1件のみ**（実際は onboardDate の約1ヶ月前からデータが存在）。826件中1件（0.1%）。ETHBTC は ESTABLISHED としてラベルされており、CRYPTO_NEW の集計には影響しない。

## 11. 生存者バイアス

### Binance では「消えかけ」が観測できる（MEXCとの決定的な違い）

MEXC の `contract/detail` は現存契約しか返さず、上場廃止銘柄は完全に不可視だった。
Binance の `exchangeInfo` は **`status: SETTLING`（決済＝上場廃止プロセス中）を123件返す**。

| status | n | 勝率 | 平均PnL | t | グリッド プラス率 | セル平均 |
|---|---|---|---|---|---|---|
| TRADING | 703 | 63.4% | +2.43% | 3.31 ★ | 100% | +1.49% |
| **SETTLING** | **123** | **74.0%** | **+6.54%** | **3.82 ★** | 100% | **+3.58%** |

**上場廃止に向かう契約は、通常契約の2.7倍の期待値でショートが刺さる。**

### これが意味すること

**バイアスの向きが確定した。** 上場廃止される銘柄＝ショートの大勝ち銘柄である。
そして**完全に上場廃止された契約は `exchangeInfo` からも消える**。つまり:

- 我々が見ているのは「生き残った銘柄 + 消えかけの銘柄」
- **本当に消えた銘柄（＝最も良いショート対象だったはず）は観測できていない**
- したがって **実測された期待値は、真の値より過小評価されている可能性が高い**

MEXC のレポートでは「バイアスの向きすら判定できない」と書いたが、**Binance の SETTLING データはその向きを示す証拠になる**（ただし SETTLING は「消えかけ」であって「消えたもの」ではないため、これはバイアスの**下限**を示すに過ぎない）。

### 未測定のリスク（重要）

**バイアスが逆に働く可能性も否定できない。** 上場廃止に向かう銘柄は流動性が枯れ、スプレッドが拡大し、板が薄くなる。
**バックテスト上の +6.54% が、実際に約定・保有できたかは、1時間足のOHLCからは判定できない。** これは推測ではなく「測定していない」という事実である。

## 14. モデル化していないもの（結果を実弾に使う前に必ず潰すこと）

**以下は測定していない。ここに書かれた期待値は、これらを無視した上澄みである。**

1. **ファンディングコスト（最大の穴）。** Binance 無期限は8時間ごとにファンディングが発生する。7日保有なら21回、14日なら42回。新規上場銘柄はファンディングが極端に振れることがあり、**7〜14日の保有では期待値と同じオーダーの損益になりうる**。ショートが受け取る側か払う側かも銘柄・時期で変わる。**今回の +3.04% は、この項を完全に無視している。**
2. **スリッページと板の厚み。** 1時間足のOHLCで「SLに触れた/TPに触れた」を判定しているが、実際にその価格で約定できたかは分からない。特に SETTLING 銘柄（+6.54%）は流動性が枯れている可能性が高い。
3. **完全に上場廃止された契約。** `exchangeInfo` から消えており、観測不能（§11）。
4. **借入・証拠金コスト、強制決済リスク。** SETTLING 銘柄は決済期日に強制クローズされる。
5. **同一足での SL/TP 同時ヒット**は 0件だったが、これは1時間足の解像度に依存する。分足で見れば発生する可能性がある。

## 15. 結論

**Binance の新規上場ショートには、2019〜2025H1 にかけて明確で再現性のあるエッジが存在した。**
n=826、期待値 +3.04%（t=4.49）、735セルのグリッドで **97%がプラス、最悪セルでも -0.26%**。
パラメータをどう振ってもほぼ負けない — これは MEXC（プラス46%、平均ゼロ）とは質的に異なる。

**その差の正体は踏み上げの尾の太さである。** MEXC では MAE p90 が 86% に達し、稀な暴騰がすべてを飲み込んでいた。Binance では 65% に収まり、中央値だけでなく平均もプラスに保たれた。

**現時点で実弾を入れる根拠は無い。** 加えて、ファンディングコストを一切モデル化していない（§14）ため、**過去のエッジですら、実装可能だったかは未検証である。**

## 16. ファンディングコストの織り込み

### 16.1 FR API の実測仕様

| 項目 | 実測値 |
|---|---|
| エンドポイント | `GET /fapi/v1/fundingRate?symbol=&startTime=&limit=` |
| レスポンス | `{symbol, fundingTime(ms), fundingRate(文字列), markPrice}` の配列 |
| **1リクエスト上限** | **500件**（`limit=1000` を投げても**黙って500件に切られる**。`limit=1001` はエラー） |
| 500件のカバー範囲 | 8h間隔で166日、4h間隔で83日 → **31日ぶんは1リクエストで足りる** |
| 遡れる範囲 | 2019-09-10（BTCUSDT）。kline最古(2019-09-08)とほぼ一致 |
| レート制限 | **`x-mbx-used-weight-1m` ヘッダが返らない**（監視不能） |
| **実測した罠** | **300ms間隔で叩くと HTTP 403（WAFバン）**。429ではない。600msに落として解消 |

**取得結果: 826/826 成功、失敗0。**

### 16.2 ファンディング間隔は8時間ではない（重大な訂正）

`GET /fapi/v1/fundingInfo` を叩くと、**723銘柄が「デフォルト以外」の間隔**を持っていた。

```
fundingIntervalHours の分布: { 1時間: 3, 4時間: 439, 8時間: 281 }
```

**439銘柄が4時間間隔**である。§14 で「8時間ごと、7日で21回」と書いたのは**誤り**で、多くの銘柄では7日間に42回発生する。

なお `fundingInfo` が返すのは**現在の設定**であり過去の間隔ではない（8h→4hに変更された銘柄がありうる）。
そのため実装では **`fundingInfo` を使わず、実際の `fundingTime` 系列を取得し、保有期間 `(entryTime, exitTime]` に入るイベントを直接合計した。** 間隔を仮定しないので、変更履歴があっても正しい。

実測された保有中のFR発生回数: **中央値16回、最大169回**。

### 16.3 符号の検算（最重要・間違えやすい）

**Binance の規約: `fundingRate > 0` → ロングがショートに払う → ショートは受け取る（PnLに加算）**

したがって **ショートのFR損益 = `+ Σ(fundingRate)`**。コストではなく、多くの場合**収入**になる。

実測での検算:

```
SNOWUSDT   FR回数=3  ΣFR=+0.0157%  (ショート受取)
   価格PnL(手数料込) = +0.88%  →  FR込み = +0.89%   差 = +0.0157%   ✓ 符号は正しい

VRTUSDT    FR回数=3  ΣFR= 0.0000%
   価格PnL = -1.82%  →  FR込み = -1.82%   差 = 0.0000%             ✓ ゼロは変化なし
```

## 17. 銘柄の出自分類（A / B / C / U / D）

### 17.1 動機

`onboardDate` は「銘柄が世に出た日」ではない。少なくとも3種類が混在している:

- **A**: Binance が初上場の完全新規銘柄
- **B**: 他CEXで既に流通していた銘柄の追加上場
- **C**: 既存銘柄の Perp 化（現物は長く存在、Perpだけ新規）
- **D**: TRADIFI（株式トークン）— 原資産は何十年も前から存在
- **U**: **判定不能**（調べる手段が無い）

### 17.2 分類に使ったシグナル（上場前の情報のみ）

**原則: 上場後の値動き（pump24・ボラティリティ等）は分類に使わない。** それは説明しようとしている結果そのものであり、循環論法になる。**上場より前に他所に存在した形跡があるか**だけで分類した。

| シグナル | 内容 | カバー率 |
|---|---|---|
| **1. Binance SPOT** | `api/v3/klines?startTime=0&limit=1` で現物の初ローソクを取得 | **409 / 696** |
| **2. MEXC** | 本セッションで取得済みの MEXC データ（t0優先、無ければcreateTime） | **553 / 696** |

分類ルール:

```
C : 現物初値 <= Perp t0 − 30日              （Binance現物に長く存在 → Perp化）
B : Cでない かつ MEXC t0 <= Binance t0 − N日  （他所で先行）
A : 上記いずれの形跡も無い
U : 現物ペアも無く、MEXCにも無い             （調べる手段が無い）
D : contractType == TRADIFI_PERPETUAL
```

**→「A（完全新規）の直近だけプラスなら本物のエッジ」という仮説は成立しない。A も死んでいる。**

### 17.8 A と B の分離には意味が無かった

**B閾値を 1日 / 7日 / 30日 と振っても、結論（Aの直近が死んでいる）は一切変わらなかった。**
これは指示された判定基準そのものであり、**A/B の区別は情報を持っていない**ことを意味する。
今後は A+B を「上場時点で新しかった銘柄」として統合してよい。

## 18. 【重要な訂正】先読みバイアス — U と SETTLING の高期待値は使えない

### 18.1 何が起きているか

§17.6 で **U（判定不能）が全クラス中で最強**に見える（+7.49%, t=4.44★, グリッド100%）。
**これは罠である。**

**U の定義は「*今日*、Binance現物にペアが無く、MEXCにも無い」。** つまり **2026年7月時点の情報**で分類している。
2021年には現物ペアがあったが**その後上場廃止された銘柄は、今日から見ると U に落ちる**。

**U は実質的に「後に死んだ銘柄」を選んでいる = 完全な先読み（look-ahead）バイアスである。**
上場した瞬間に、その銘柄が将来 U になるかは知りようがない。**この +7.49% は取引できない。**

A / B / C は「上場**前**に他所に存在した**証拠**」で定義されており、先読みではない。
**U だけが「今日の*不在*」で定義されている。** 「証拠の不在」は「不在の証拠」ではなく、しかも参照している時点が未来である。

→ **U を独立クラスとして分離したのは正解だった。** A に畳み込んでいたら、A の数字が偽って良く見えていた。

### 18.2 同じ理由で §11 の SETTLING を訂正する

§11 で `status: SETTLING`（上場廃止プロセス中）が **+6.54%（t=3.82）** と報告し、
「上場廃止に向かう契約はショートが刺さる」と書いた。**この数字自体は正しいが、戦略としては使えない。**

**上場した時点では、その契約が将来 SETTLING になるかは分からないからである。** これも先読みである。

**訂正: SETTLING の +6.54% は「生存者バイアスの向きを示す証拠」としては有効だが、「エントリー条件」にはならない。**
§11 の「バイアスの向きが確定した（消えた銘柄ほど良いショート対象 → 実測値は過小評価の可能性）」という結論は維持する。
「+6.54% を狙って SETTLING 銘柄をショートする」という読み方は誤りである。

### 18.3 教訓

**分類・フィルタに使う情報は、必ず「エントリー時点で知り得たもの」でなければならない。**

今回、以下は安全だった:
- pump24 / ボラティリティ / 出来高（エントリー前24時間の実データ）
- Binance現物の初ローソク（上場より前の事実）
- MEXC の初取引時刻（上場より前の事実）

以下は**先読みで汚染されている**:
- U（今日の不在で定義）
- SETTLING（将来の状態）
- 「現在も上場中か」に依存するあらゆる分類

## 19. 追記を踏まえた最終結論

1. **FR はエッジを壊さない。** コストは平均 -0.20%（ショートは65%のトレードでFRを受け取る側）。
   §14 の「FRが最大の穴」という警告は**外れだった**。この懸念は消滅した。

2. **しかしエッジ自体が 2025H2 に消えている。** FR を織り込んでも:
   2025H1 +5.17%(t=2.81★) → **2025H2 +0.34%(t=0.17)** → **2026H1 +0.20%(t=0.15)**

3. **銘柄の出自で分けても救えない。** 本命だった **A（Binance初上場・完全新規）も 2025H2 でマイナスに反転**
   （-1.69%, n=87）。2026H1 は n=32 で判断不能。**A/B の分離自体に情報が無かった**（閾値で結論が変わらない）。

4. **U(+7.49%) と SETTLING(+6.54%) の高期待値は先読みバイアスであり、実弾では再現しない。**

**Binance の新規上場ショートも、MEXC と同じく「今は効かない」。**
過去（2019〜2025H1）には FR 込みでも明確なエッジが実在したが、**2025H2 以降は消えている。**
両取引所で同時期に消えており、同時期に新規上場の主役が株式トークンへ移行している（Binance 2026H1: 71%、MEXC: 48%）。

**実弾投入の根拠は無い。**
