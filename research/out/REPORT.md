## 1. 目的（検証した仮説）

**「MEXCの新規上場銘柄は、上場直後に適当にショートすれば勝てる」**

この仮説を、フィルターを一切かけない全数データで検証する。
副次的に、以下も検証対象とした。

- 初動でポンプした銘柄ほどショートが効くのか（pump24仮説）
- パラメータ（エントリー時刻・SL・TP・保有期間）に最適領域は存在するか
- 過去に発生していたバグ（存在しない価格でのエントリー判定、kline空での永久リトライ）の原因

### アンカーの定義

**t0 = 実際に返ってきた最初のローソクの時刻**。`createTime` はアンカーに使っていない（理由は §3）。

各銘柄について `[t0 - 6h, t0 + 30d]` を取得。
`time > now` の未来行は切り捨てた（**35銘柄・計8,512本**が該当）。切らなかった場合、直近上場銘柄のリターンが架空の値で埋まる。

### レスポンス構造（実測）

`data` は配列ではなく列指向オブジェクト。`time` は Unix **秒**。

```
data = { time[], open[], close[], high[], low[], vol[], amount[],
         realOpen[], realClose[], realHigh[], realLow[] }
```

## 3. 発見した過去バグの正体

**`createTime` は「契約が作られた時刻」であって「取引が始まった時刻」ではない。**

788件中、`createTime` と t0（初ローソク）のズレ:

| ズレ | 件数 | 割合 |
|---|---|---|
| **1日以上** | **109** | **13.8%** |
| （うち）7日以上 | 16 | — |
| （うち）90日以上 | 13 | — |

最悪ケース（365日ユニバースでの実測）:

| symbol | createTime | t0（初ローソク） | 差 |
|---|---|---|---|
| `EBAYSTOCK_USDT` | 2025-08-01 | 2026-06-09 | **312.2日** |
| `APLDSTOCK_USDT` | 2025-08-01 | 2026-05-22 | 293.8日 |
| `NFLXSTOCK_USDT` | 2025-09-11 | 2026-06-09 | 271.2日 |
| `APPSTOCK_USDT` | 2025-09-17 | 2026-05-28 | 252.6日 |
| `BMNRSTOCK_USDT` | 2025-08-13 | 2026-03-17 | 216.2日 |
| `AIA_USDT` | 2025-09-17 | 2026-01-20 | 125.1日 |
| `ASTER_USDT` | 2025-09-17 | 2025-09-19 | 2.0日 |

ワースト15は `AIA_USDT` を除きすべて `*STOCK_USDT`。株式トークンは「契約だけ先に作り、数ヶ月後に取引開始」が常態化している。

**帰結**: `createTime` を上場時刻として扱うと、これら109銘柄では**まだ1本もローソクが存在しない時刻でエントリー判定に入る**。kline が空で返るため、価格が取得できず、リトライが終わらない。これが観測されていた不具合の挙動と一致する。

**対策**: アンカーは必ず初ローソクの時刻を使う。`createTime` は使わない。

## 5. 生存者バイアス

**このバイアスの大きさは、使用したAPIでは測定不能である。**

`contract/detail` は**現在上場中の契約しか返さない**。上場廃止された契約は存在しない。したがって「当時MEXCが実際に何本上場したか」を知る手段が、このエンドポイントには無い。

コホート別の生存数（＝現在も上場している契約数）:

| コホート | 生存数 | t0からの経過 |
|---|---|---|
| 2023H2 | 25 | 約33ヶ月 |
| 2024H1 | 46 | 約27ヶ月 |
| 2024H2 | 76 | 約21ヶ月 |
| 2025H1 | 139 | 約15ヶ月 |
| 2025H2 | 169 | 約9ヶ月 |
| 2026H1 | 277 | 約2ヶ月 |

**この減少を上場廃止の証拠として読むことはできない。** 減少には「上場廃止による目減り」と「近年の上場ペース増加」（2026年6月だけで97本作成）が混在しており、この2つは detail.json だけでは分離できない。前セクションで示した月別件数も、同じ生存者集合を数え直したものにすぎない。

言えるのは方向性のみ:

- MEXCは出来高の枯れた契約を上場廃止する。
- ポンプ後に暴落して流動性が枯れた銘柄＝**ショートの大勝ち銘柄**が優先的に消えている可能性がある。この場合、2024H2/2025H1 の +5% は過小評価であり、真の値はより良い。
- 一方、上場廃止に至る流動性枯渇局面ではスプレッドが拡大し、実際に約定・保有できたかは不明。この場合、バックテスト上の利益は実現不可能。

**どちらに転ぶかは、このデータでは判定できない。** これは「保守的に見積もれば済む」種類の問題ではなく、「検証できない」種類の問題である。

## 6. 結論

1. **「新規上場を無差別にショートすれば勝てる」は、直近1年（485件）のデータでは成立しない。** 期待値 -0.00%（n=482）。パラメータを735通り振っても、プラスのセルは46%、全体平均 +0.004%。

2. **層別化しても、統計的に信頼できる正の期待値を持つ層は1つも見つからなかった。** pump24 / ボラティリティ / 出来高 / シンボル名、いずれの軸でも。

3. **「初動で100%以上ポンプした銘柄を狙う」仮説は棄却。** サンプルを倍増（23→45件）して検証した結果、t=1.86 で有意でなく、符号安定性（67%）は全ユニバース（84%）を下回った。

4. **3年に広げると勝てるように見えるが、それは 2024H2〜2025H1 の2コホートだけの現象であり、2025H2 で符号が反転し、直近 2026H1（n=277）では消えている。**

5. **エッジの消失は、株式トークンによるユニバースの質的変化（STOCK比率 0% → 48%）とタイミングが一致する。** 「新規上場」というカテゴリの意味そのものが、この1年で変質した。

6. **生存者バイアスの大きさは測定不能。** この不確実性を残したまま実弾を入れるべきではない。

7. **技術的成果として、過去バグの原因は特定できた。** `createTime` ≠ 取引開始時刻（13.8%が1日以上ズレ、最悪312日）。アンカーを初ローソクに変更すれば解消する。

**現時点の推奨: この戦略に資金を投入しない。** パラメータ探索や層別化をこれ以上続けても、同じ結論に到達する。

前進するために必要なのは最適化ではなく、**上場廃止された契約を含む完全なユニバース**である。取りうる手段は、外部データソースの導入、または `contract/detail` の日次スナップショットを今から蓄積して自前で構築すること。それ無しでの最適化は、生存者だけを見て「みんな生き残っている」と結論することと同じ。
